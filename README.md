# Yii2 Project

## 🚀 Развертывание и запуск

### Вариант 1: Docker (Рекомендуемый)

Для работы требуются установленные [Docker](https://www.docker.com/) и [Docker Compose](https://docs.docker.com/compose/).

Конфигурация Docker в этом проекте требует передачи `UID` и `GID` текущего пользователя при сборке, чтобы избежать проблем с правами доступа к файлам.

1. **Сборка и запуск контейнеров (Production)**

   Эта команда соберет образ приложения, скопировав в него исходный код, и запустит все сервисы в фоновом режиме. Зависимости Composer устанавливаются автоматически во время сборки.
   ```bash
   docker-compose -f docker-compose.prod.yml up -d --build
   ```

2. **Применение миграций**

   После первого запуска контейнеров необходимо применить миграции базы данных:
   ```bash
   docker-compose -f docker-compose.prod.yml exec php ./yii migrate
   ```

Проект будет доступен по адресу, указанному в вашей конфигурации (обычно `http://localhost`).

### 🛠 Вариант 2: Ручная установка (Без Docker)

**Системные требования:**
- PHP 8.1
- Расширения PHP: `pdo_mysql`, `intl`, `zip`, `gd`, `opcache`
- Composer
- MySQL или MariaDB
- Веб-сервер (Nginx или Apache)

1. **Установка зависимостей**
   Перейдите в директорию приложения `app` и выполните установку:
   ```bash
   cd app
   composer install
   ```

2. **Настройка базы данных**
   - Создайте базу данных в вашей СУБД.
   - Пропишите параметры подключения в файле конфигурации (обычно `config/db.php` или `common/config/main-local.php`).

3. **Запуск**
   Примените миграции и запустите встроенный веб-сервер (или настройте Nginx/Apache на папку `app/web`):
   ```bash
   php yii migrate
   php yii serve
   ```
